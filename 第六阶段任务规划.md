# SecRAG Agent 第六阶段任务规划

## 一、阶段目标

第六阶段目标是加入威胁情报、历史记忆和多源融合决策，让系统从“单条日志分析”升级为“带上下文的安全研判”。

第五阶段完成后，系统已经具备：

```text
WAF 日志
  -> 攻击识别
  -> RAG 安全知识增强
  -> Markdown 报告
```

第六阶段要新增：

```text
source_ip / domain / user_agent
  -> ThreatIntelAgent
  -> 本地威胁情报库 / 可选外部 API
  -> Memory
  -> 历史行为统计
  -> 决策融合
```

本阶段重点是让告警更像 SOC 场景中的研判结果，而不是孤立判断。

## 二、阶段价值

加入威胁情报和记忆后，系统可以回答：

```text
这个 IP 是否多次攻击？
是否命中过多个攻击类型？
是否是扫描器或代理节点？
同一来源过去是否攻击过其他路径？
当前风险是否应该因为历史行为而提高？
```

这会显著增强项目的企业级安全运营味道。

## 三、核心链路

```text
SecurityEvent
  -> ThreatIntelAgent
  -> MemoryStore
  -> DecisionAgent
  -> SecurityAlert
```

多源信息融合：

```text
日志证据
RAG 知识证据
威胁情报证据
历史行为证据
  -> 风险评分
  -> 风险等级
  -> 响应建议
```

## 五、模块职责

### 1. threat_intel_agent.py

负责根据 IP、域名、User-Agent 查询威胁情报。

第一版建议只做本地 mock：

```text
45.67.89.10 -> scanner / sqlmap / high
8.8.8.8 -> unknown / low
9.9.9.9 -> suspicious / medium
```

后续再预留真实 API 适配器。

### 2. local_intel_store.py

负责读取本地威胁情报文件。

数据文件：

```text
backend/app/data/intel/threat_ip_mock.json
```

建议字段：

```text
ip
reputation
tags
risk_score
source
description
```

### 3. event_memory.py

负责记录历史事件。

MVP 可以使用 Redis 存储：

```text
ip:{source_ip}:events
ip:{source_ip}:attack_count
ip:{source_ip}:last_seen
```

也可以先使用 Redis Stream 中已有历史告警做统计。

### 4. decision_agent.py

扩展风险评分逻辑。

建议融合维度：

```text
攻击类型基础分
WAF 命中规则
RAG 知识严重性
威胁情报风险分
历史攻击次数
自动化扫描器特征
```

### 5. orchestrator.py

编排顺序调整为：

```text
LogParserAgent
  -> RAGAgent
  -> ThreatIntelAgent
  -> Memory
  -> DecisionAgent
  -> SecurityAlert
```

## 六、推荐开发顺序

### Step 1：定义威胁情报模型

文件：

```text
backend/app/intel/schemas.py
```

建议模型：

```text
ThreatIntelResult
```

字段：

```text
source_ip
reputation
risk_score
tags
source
description
```

### Step 2：准备本地 mock 情报库

文件：

```text
backend/app/data/intel/threat_ip_mock.json
```

先写 3 到 5 个 IP 即可。

### Step 3：编写 ThreatIntelAgent

文件：

```text
backend/app/intel/threat_intel_agent.py
```

输入：

```text
source_ip
```

输出：

```text
ThreatIntelResult
```

### Step 4：编写 EventMemory

文件：

```text
backend/app/memory/event_memory.py
```

第一版先实现：

```text
record_event(event)
get_ip_summary(source_ip)
```

### Step 5：扩展 DecisionAgent

让风险评分受到威胁情报和历史行为影响。

示例：

```text
命中 SQL Injection 基础分 85
source_ip 为 scanner +10
过去 24 小时攻击次数 > 3 +5
最终 critical
```

### Step 6：扩展报告

在 `report_markdown` 中新增：

```text
威胁情报
历史行为
综合研判
```

## 七、验收标准

满足以下条件，即认为第六阶段完成：

```text
ThreatIntelAgent 可以返回 IP 情报
本地 mock 情报库可以正常读取
Memory 可以记录和查询 IP 历史行为
DecisionAgent 可以融合情报和历史行为调整风险
SecurityAlert evidence 中包含威胁情报证据
report_markdown 中包含威胁情报和历史行为小节
同一 IP 多次攻击时，风险判断能体现上下文
```

## 八、暂不实现

第六阶段先不做：

```text
真实 AbuseIPDB / VirusTotal API 强依赖
复杂图数据库攻击链
用户画像系统
自动封禁执行
多租户情报隔离
```

真实 API 可以作为可选增强，不影响 MVP。

## 九、风险点

### 1. 外部 API 不稳定

演示项目优先使用本地 mock 情报，真实 API 只做适配预留。

### 2. 风险评分过于随意

建议把评分规则写清楚，哪怕简单，也要可解释。

### 3. 历史数据重复计算

MVP 可以接受简单统计，后续再做去重和时间窗口。
