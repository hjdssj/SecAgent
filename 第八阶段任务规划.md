# SecRAG Agent 第八阶段任务规划

## 一、阶段目标

第八阶段目标是工程化收尾、演示固化和项目包装，让 SecRAG Agent 成为一个可以稳定启动、完整演示、方便讲解的项目。

第七阶段完成后，系统已经具备：

```text
WAF 接入
Redis 事件流
Agent 分析链路
RAG 知识增强
威胁情报
前端 SOC 控制台
```

第八阶段要完成：

```text
一键启动
演示脚本
测试用例
README
架构图
工作日志总结
GitHub 项目整理
```

## 二、阶段价值

这一阶段决定项目是否“能讲清楚、能跑起来、能交付”。

完成后，别人打开项目应该能快速理解：

```text
这是做什么的
系统怎么启动
怎么产生攻击流量
怎么看分析结果
每个模块职责是什么
项目亮点是什么
```

## 三、最终演示链路

```text
docker compose up
  -> Redis + WAF
uvicorn app.main:app
  -> FastAPI
python -m app.collector.waf_log_collector
  -> WAF 日志采集
python -m app.services.event_consumer
  -> 告警分析
npm run dev
  -> SOC 前端
scripts/simulate_attack.py
  -> 产生攻击请求
浏览器查看 SOC Dashboard
```

## 七、推荐开发顺序

### Step 1：整理 README

先把项目定位和启动方式写清楚。

重点突出：

```text
企业级 SOC 场景
WAF 实时日志接入
RAG 安全知识增强
多 Agent 分析
结构化告警与 Markdown 报告
```

### Step 2：编写 demo guide

文件：

```text
docs/demo_guide.md
```

要求别人照着文档能跑通演示。

### Step 3：补测试

优先写 parser 和 agent 测试。

建议命令：

```text
pytest
python -m compileall backend/app
```

### Step 4：准备演示脚本

文件：

```text
scripts/simulate_attack.py
```

支持：

```text
normal
sqli
xss
path_traversal
all
```

### Step 5：整理工作日志

在 `工作日志.md` 中按阶段总结：


## 九、暂不实现

第八阶段先不做：

```text
生产级部署
Kubernetes
CI/CD 全流程
用户登录权限
企业级告警闭环工单
真实封禁执行
```

这些可以作为项目后续扩展方向。

## 十、推荐最终项目亮点写法

```text
SecRAG Agent 是一个面向 SOC 场景的实时安全分析与响应系统。
系统通过接入 Nginx + ModSecurity + OWASP CRS 产生的 WAF 日志，
使用 Redis Stream 构建事件流，
并结合多 Agent、RAG 安全知识库、威胁情报和历史记忆，
自动完成攻击识别、风险评估、ATT&CK 映射、证据归纳和 Markdown 报告生成。
```
