# SecRAG Agent 测试方法

## 一、为什么可以执行 uvicorn 命令

后端使用 FastAPI 编写，但 FastAPI 本身只是 Web 框架。真正把 FastAPI 应用启动成 HTTP 服务的是 Uvicorn。

启动命令：

```bash
uvicorn app.main:app --reload --port 8000
```

含义如下：

```text
uvicorn
```

启动 ASGI 服务。

```text
app.main:app
```

表示从 `backend/app/main.py` 这个模块中，找到名为 `app` 的 FastAPI 实例。

它对应代码中的：

```python
from fastapi import FastAPI

app = FastAPI()
```

其中：

```text
app.main
```

表示 Python 模块路径：

```text
backend/app/main.py
```

冒号后面的：

```text
:app
```

表示 `main.py` 文件里的变量：

```python
app = FastAPI(...)
```

```text
--reload
```

表示开发模式自动重载。修改代码后，服务会自动重启。

```text
--port 8000
```

表示服务运行在 8000 端口。

注意：这个命令建议在 `backend/` 目录下执行。

```bash
cd backend
uvicorn app.main:app --reload --port 8000
```

如果在项目根目录执行，模块路径可能需要改成：

```bash
uvicorn backend.app.main:app --reload --port 8000
```

## 二、启动后端服务

进入后端目录：

```bash
cd backend
```

安装依赖：

```bash
pip install -r requirements.txt
```

启动服务：

```bash
uvicorn app.main:app --reload --port 8000
```

启动成功后，浏览器打开：

```text
http://127.0.0.1:8000/docs
```

这是 FastAPI 自动生成的 Swagger 接口文档页面。

## 四、测试安全日志分析接口

在 Swagger 页面中找到：

```text
POST /api/analyze
```

点击：

```text
Try it out
```

然后在请求体中输入测试 JSON。

### 1. SQL 注入测试

请求体：

```json
{
  "event_id": "evt-demo-0001",
  "source_ip": "45.67.89.10",
  "method": "GET",
  "url": "/login?id=1' OR '1'='1",
  "path": "/login",
  "query": "id=1' OR '1'='1",
  "status": 403,
  "user_agent": "sqlmap/1.7",
  "waf_rule_id": "942100",
  "waf_message": "SQL Injection Attack Detected",
  "raw_log": "GET /login?id=1' OR '1'='1 HTTP/1.1"
}
```

预期重点字段：

```text
attack_type: SQL Injection
risk_level: high
source_ip: 45.67.89.10
target: /login
evidence: 应包含 SQL 注入特征、WAF 规则命中、sqlmap 特征
recommendations: 应包含参数化查询检查、WAF 规则加固、源 IP 回溯
```

### 2. XSS 测试

请求体：

```json
{
  "event_id": "evt-demo-0002",
  "source_ip": "8.8.8.8",
  "method": "GET",
  "url": "/search?q=<script>alert(1)</script>",
  "path": "/search",
  "query": "q=<script>alert(1)</script>",
  "status": 403,
  "user_agent": "Mozilla/5.0",
  "waf_rule_id": "941100",
  "waf_message": "XSS Attack Detected",
  "raw_log": "GET /search?q=<script>alert(1)</script> HTTP/1.1"
}
```

预期重点字段：

```text
attack_type: XSS
risk_level: medium 或 high
source_ip: 8.8.8.8
target: /search
evidence: 应包含 XSS 特征和 WAF 规则命中
```

### 3. 路径穿越测试

请求体：

```json
{
  "event_id": "evt-demo-0003",
  "source_ip": "9.9.9.9",
  "method": "GET",
  "url": "/download?file=../../etc/passwd",
  "path": "/download",
  "query": "file=../../etc/passwd",
  "status": 403,
  "user_agent": "Mozilla/5.0",
  "waf_rule_id": "930120",
  "waf_message": "Path Traversal Attack Detected",
  "raw_log": "GET /download?file=../../etc/passwd HTTP/1.1"
}
```

预期重点字段：

```text
attack_type: Path Traversal
risk_level: high
source_ip: 9.9.9.9
target: /download
evidence: 应包含路径穿越特征和 WAF 规则命中
```

## 五、如何判断第一阶段测试通过

第一阶段不要求接入真实 WAF，也不要求 RAG 检索。

只要满足下面条件，就算第一阶段后端分析闭环通过：

1. `GET /api/health` 能返回服务正常。
2. `POST /api/analyze` 能接收一条安全事件 JSON。
3. SQL 注入样例能返回 `SQL Injection`。
4. XSS 样例能返回 `XSS`。
5. 路径穿越样例能返回 `Path Traversal`。
6. 返回结果中包含风险等级、证据列表和处置建议。

## 六、常见问题

### 1. 找不到 uvicorn

说明依赖没有安装。

执行：

```bash
pip install -r requirements.txt
```

### 2. 报错 No module named app

通常是启动目录不对。

请确认你在 `backend/` 目录下执行：

```bash
uvicorn app.main:app --reload --port 8000
```

### 3. 修改代码后接口结果没变

确认启动命令里有：

```text
--reload
```

如果仍然没变，手动停止服务后重新启动。

### 4. 返回 422

说明请求 JSON 字段类型不符合 Pydantic 模型。

重点检查：

- `status` 应该是数字，例如 `403`
- `confidence` 不需要输入，由后端生成
- JSON 字符串必须使用双引号
