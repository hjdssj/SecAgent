# 第十二阶段任务规划：告警持久化与状态更新

## 一、阶段定位

第十二阶段的目标是把 SecRAG Agent 的告警从“实时分析结果”升级为“可长期查询、可人工复核、可状态闭环的告警档案”。

当前系统已经具备：

```text
WAF 日志接入
Redis Stream 实时事件流
攻击识别
真实 RAG 检索
成本可控分析模式
企业上下文 RAG
AutoTriage 自动分流
前端 SOC 展示
```

但目前告警状态主要存在于 `SecurityAlert` 响应和 Redis 实时流中，还缺少长期持久化。

本阶段要补齐：

```text
数据库 alerts 表
告警生成后入库
从数据库查询最近告警
按状态筛选
按是否需要人工复核筛选
更新告警状态
更新分析备注
前端人工复核操作
```

## 二、存储边界

### Redis 存什么

Redis 继续负责：

```text
实时事件流
短期记忆
缓存
去重
```

例如：

```text
security:events
security:alerts
memory:ip:{source_ip}
intel:ip:{source_ip}
rag:cache:{query_hash}
dedup:alert:{fingerprint}
```

### 数据库存什么

数据库负责：

```text
告警最终状态
AutoTriage 结果
人工复核状态
分析备注
处理人
处理时间
复盘查询
```

一句话：

```text
Redis 管实时，数据库管闭环。
```

## 八、API 设计

### 1. 查询最近告警

增强现有接口：

```text
GET /api/alerts/recent
```

查询参数：

```text
count
status
requires_human_review
```

示例：

```text
GET /api/alerts/recent?count=20
GET /api/alerts/recent?status=needs_review
GET /api/alerts/recent?requires_human_review=true
GET /api/alerts/recent?status=needs_review&requires_human_review=true
```

返回：

```text
list[SecurityAlert]
```

### 2. 更新告警状态

新增：

```text
PATCH /api/alerts/{alert_id}/status
```

请求体：

```json
{
  "status": "resolved",
  "analyst_note": "WAF 已阻断，未发现后端异常，已通知账号中心负责人复核。",
  "handled_by": "analyst"
}
```

返回：

```text
SecurityAlert
```

错误：

```text
404 - alert_id 不存在
422 - status 非法
```

### 3. 单条详情

本阶段可选：

```text
GET /api/alerts/{alert_id}
```

如果前端暂时从列表选中告警展示详情，可以先不做。

## 九、后端写入链路

### event_consumer

当前：

```text
Redis security:events
  -> event_consumer
  -> SecurityAnalysisOrchestrator
  -> Redis security:alerts
```

目标：

```text
Redis security:events
  -> event_consumer
  -> SecurityAnalysisOrchestrator
  -> Redis security:alerts
  -> DB alerts 表
```

### /api/analyze

当前：

```text
POST /api/analyze
  -> SecurityAnalysisOrchestrator
  -> SecurityAlert
```

目标：

```text
POST /api/analyze
  -> SecurityAnalysisOrchestrator
  -> SecurityAlert
  -> DB alerts 表
```

### /api/alerts/recent

当前：

```text
从 Redis security:alerts 读取
```

目标：

```text
优先从 DB alerts 表读取
Redis 可作为 fallback
```
