# 第十一阶段任务规划：企业上下文 RAG 与 AutoTriage 自动分流

## 一、阶段定位

第十一阶段的目标是把 SecRAG Agent 从“安全知识增强告警”升级为“结合企业上下文自动分流的 SOC 告警分析系统”。

本阶段不做数据库持久化，也不做复杂人工工单，而是先解决一个更关键的问题：

```text
系统如何知道这个告警和企业业务有什么关系？
```

也就是让系统可以自动回答：

```text
这个接口属于哪个业务？
负责人是谁？
目标资产是否高危？
源 IP 是否内部扫描器？
当前 WAF 规则是 block 还是 detect？
是否处于变更窗口？
是否需要人工复核？
```

## 六、自动分流规则

当前规则：

```text
内部扫描器 + 风险低于 high -> auto_triaged / auto_close
critical 资产 + high 以上风险 -> needs_review / human_review_required
critical 风险分 -> needs_review / human_review_required
high 风险分 -> needs_review / notify_owner
WAF block + 风险低于 deep 阈值 -> auto_triaged / observe_only
上下文不足 -> needs_review / human_review_required
```
