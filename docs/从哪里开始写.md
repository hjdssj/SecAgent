# SecRAG Agent 从哪里开始写

## 你的第一阶段目标

第一阶段不要急着接 WAF、Milvus、LlamaIndex、前端大屏。

先完成一个最小闭环：

```text
输入一条安全日志 JSON
  ↓
LogParserAgent 识别攻击特征
  ↓
DecisionAgent 判断风险等级
  ↓
FastAPI 返回结构化告警
```

现在仓库里已经搭好了这个最小骨架。

## 第一步：先理解数据流

从这个接口开始看：

```text
backend/app/api/analyze.py
```

它接收一个 `SecurityEvent`，然后调用：

```text
backend/app/agents/orchestrator.py
```

编排器会依次调用：

```text
backend/app/agents/log_parser_agent.py
backend/app/agents/decision_agent.py
```

最后返回：

```text
backend/app/models/alert.py
```

## 第二步：先跑起来

进入后端目录：

```bash
cd backend
```

安装依赖：

```bash
pip install -r requirements.txt
```

启动后端：

```bash
uvicorn app.main:app --reload --port 8000
```

打开接口文档：

```text
http://127.0.0.1:8000/docs
```

测试健康检查：

```text
GET /api/health
```

测试日志分析：

```text
POST /api/analyze
```

请求体可以用：

```text
backend/sample_event_sqli.json
```

## 第三步：你应该先改哪里

### 1. 改攻击识别规则

文件：

```text
backend/app/agents/log_parser_agent.py
```

先增加这些攻击类型：

- SQL Injection
- XSS
- Path Traversal
- Command Injection
- Brute Force
- Automated Scanner

你可以从正则规则开始写，不要一开始就让大模型判断。

原因：

- 规则检测稳定。
- 演示效果可控。
- 后续 RAG 和 LLM 负责解释、补证据和生成报告。

### 2. 改风险评分

文件：

```text
backend/app/agents/decision_agent.py
```

先把风险评分逻辑写清楚：

```text
攻击类型基础分
+ WAF 命中加分
+ 403 拦截加分
+ 自动化扫描加分
+ 威胁情报加分，后续补
```

### 3. 改输出字段

文件：

```text
backend/app/models/alert.py
```

你后续前端展示什么，这里就要有什么。

比如：

- `attack_type`
- `risk_score`
- `risk_level`
- `source_ip`
- `target`
- `evidence`
- `mitre_attack`
- `recommendations`
- `report_markdown`

## 第四步：第二个小目标

当 `/api/analyze` 能稳定返回告警后，再做第二个小目标：

```text
样例日志文件
  ↓
日志解析器
  ↓
批量调用 analyze
  ↓
生成多条告警
```

你需要新增：

```text
backend/app/collector/modsecurity_parser.py
scripts/replay_sample_logs.py
data/sample_logs/
```

这个阶段还不用 Redis。

## 第五步：第三个小目标

再接 Redis Stream：

```text
Log Collector
  ↓
Redis Stream: security:events
  ↓
FastAPI Consumer
  ↓
Agent Orchestrator
  ↓
Redis Stream: security:alerts
```

你需要新增：

```text
backend/app/storage/redis_client.py
backend/app/services/event_consumer.py
backend/app/collector/waf_log_collector.py
```

## 第六步：第四个小目标

最后接 LlamaIndex RAG：

```text
攻击特征
  ↓
Query Rewrite
  ↓
LlamaIndex Retriever
  ↓
CVE / ATT&CK / OWASP CRS / 修复建议
  ↓
DecisionAgent 生成带引用的报告
```

你需要新增：

```text
backend/app/rag/index_builder.py
backend/app/rag/retriever.py
backend/app/rag/query_rewriter.py
backend/app/agents/rag_agent.py
data/knowledge_base/
```

## 推荐你今天的写法

今天只盯住这个顺序：

1. 跑通 `POST /api/analyze`。
2. 补全 `LogParserAgent` 的攻击规则。
3. 补全 `DecisionAgent` 的风险评分。
4. 新增 3 个样例 JSON：SQLi、XSS、路径穿越。
5. 再写一个日志回放脚本。

等这些稳定了，再接 RAG 和前端。

## 判断你是否完成第一阶段

如果你能做到下面这个效果，就算第一阶段完成：

输入：

```text
/login?id=1' OR '1'='1
```

输出：

```text
攻击类型：SQL Injection
风险等级：high
证据：命中 SQL 注入特征、命中 WAF 规则、User-Agent 为 sqlmap
建议：检查参数化查询、提高 WAF 规则、回溯源 IP
```

这就是后续所有能力的地基。
