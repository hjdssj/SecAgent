# 修复建议知识库

## Web 输入校验

### 适用场景
所有接收用户输入的 Web 接口，包括 query、form、JSON body、header、cookie。

### 修复建议
- 使用白名单校验输入类型、长度、范围和格式。
- 对安全敏感字段进行服务端校验，不依赖前端校验。
- 对异常输入记录审计日志。

## 数据库安全绑定

### 适用场景
SQL Injection、搜索接口、登录接口、筛选接口。

### 修复建议
- 使用参数化查询。
- 使用 ORM 的安全绑定能力。
- 避免字符串拼接 SQL。
- 数据库账号使用最小权限。

## 输出编码

### 适用场景
XSS、搜索结果页、评论区、富文本、错误页面。

### 修复建议
- HTML 正文使用 HTML entity 编码。
- HTML 属性使用属性上下文编码。
- JavaScript 字符串使用 JavaScript 编码。
- URL 参数使用 URL 编码。

## 文件访问边界

### 适用场景
Path Traversal、文件下载、图片预览、报表导出。

### 修复建议
- 使用文件 ID 代替真实文件路径。
- 将文件访问限制在固定根目录。
- 拒绝 `../`、绝对路径和特殊系统文件名。
- 对可下载文件设置白名单。

## WAF 运营动作

### 适用场景
WAF 已命中规则或安全事件需要复盘。

### 修复建议
- 保留 WAF audit log、access log 和原始请求证据。
- 回溯源 IP 在过去 24 小时内的访问行为。
- 对高危攻击临时提高相关规则阻断级别。
- 将误报样本记录到后续规则调优列表。
