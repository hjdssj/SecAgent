# 攻击特征知识库

## SQL Injection

### 攻击说明
SQL Injection 是攻击者通过请求参数插入 SQL 片段，改变后端数据库查询逻辑的攻击方式，常见于登录、搜索、详情查询等接口。

### 常见特征
- `' OR '1'='1`
- `UNION SELECT`
- `information_schema`
- `sleep()`
- `benchmark()`
- sqlmap User-Agent

### 风险影响
可能造成认证绕过、数据泄露、数据篡改，严重时可能进一步扩大到服务器控制风险。

### 修复建议
- 使用参数化查询或 ORM 安全绑定。
- 避免将用户输入直接拼接进 SQL 字符串。
- 对输入字段做类型、长度和白名单校验。
- 对 SQL 注入相关 WAF 规则开启阻断。

## XSS

### 攻击说明
XSS 是攻击者将恶意脚本注入页面，使脚本在其他用户浏览器中执行的攻击方式。

### 常见特征
- `<script>`
- `javascript:`
- `onerror=`
- `onload=`
- `alert()`

### 风险影响
可能造成 Cookie 窃取、账号劫持、钓鱼页面植入、前端操作劫持。

### 修复建议
- 对输出内容进行上下文相关编码。
- 对富文本输入使用安全白名单。
- 配置 Content-Security-Policy 降低脚本执行风险。
- 避免把未转义的用户输入直接写入 HTML。

## Path Traversal

### 攻击说明
Path Traversal 是攻击者通过 `../` 等路径跳转符访问预期目录之外文件的攻击方式，常见于下载、预览、导出接口。

### 常见特征
- `../`
- `..\\`
- `/etc/passwd`
- `boot.ini`
- `win.ini`

### 风险影响
可能造成敏感配置、源码、凭据或系统文件泄露。

### 修复建议
- 限制文件访问目录边界。
- 使用文件 ID 映射真实路径，避免用户直接控制路径。
- 对文件名做白名单校验。
- 拒绝包含路径跳转符的输入。

## Command Injection

### 攻击说明
Command Injection 是攻击者通过输入拼接系统命令，使服务端执行非预期命令的攻击方式。

### 常见特征
- `; whoami`
- `| id`
- `` `id` ``
- `$(whoami)`
- `curl` 或 `wget` 远程下载命令

### 风险影响
可能造成服务器命令执行、反弹 shell、文件窃取和横向移动。

### 修复建议
- 禁止将用户输入直接拼接到系统命令。
- 优先使用语言内置 API 替代 shell。
- 对允许的参数做严格白名单。
- 降低服务运行权限。

## Automated Scanner

### 攻击说明
Automated Scanner 是自动化扫描器或攻击工具产生的探测行为，常用于批量识别漏洞入口。

### 常见特征
- sqlmap
- nikto
- acunetix
- nessus
- 高频访问多个敏感路径

### 修复建议
- 回溯源 IP 的访问路径和访问频率。
- 对扫描源进行限速或临时封禁。
- 检查扫描命中的路径是否存在真实漏洞。
